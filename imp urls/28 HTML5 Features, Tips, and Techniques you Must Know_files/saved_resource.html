                                                                                            #5 - Not quoting is problematic, and not really a new thing. I've seen people do this for ages and it annoys me. I think it's silly and lazy. Your text editors won't color code properly, so it will be less readable and you won't catch typos as easily. HTML validators will throw errors. What if the attribute actually has a space, like in a caption? Now you need quotes. Now you're writing inconsistent markup. The only time I would possibly use this would be when nesting quotes. Instead of having to escape you could leave them out. Although, you can normally avoid 3 levels of quotes. Also, I'm pretty sure this makes the browser do extra work to figure out where the attributes actually ends, since it doesn't have an explicit delimiter. Dislike.
