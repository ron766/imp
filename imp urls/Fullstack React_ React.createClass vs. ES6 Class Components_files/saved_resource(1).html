              loopmode • a year ago

              Since I use decorator syntax via Babel, and I work with sublime text, I switched from property initializers to autobind-decorator. The reason is that sublime detects regular class methods correctly and allows you to jump to methods using Ctrl+R much like eclipse/Java Ctrl+O, while this does not work with prop-initializer arrow functions. Might seem weird to adjust your coding style to match an editor you use, but I am really happy with this solution, and together with F12 to jump into class definitions across files, this almost feels like a real IDE. Just as a tip to folks using sublime text. :)
              Plus I think readability is much better when your methods are shaped in a homogeneous way and you read a class or react component top-to-bottom.
